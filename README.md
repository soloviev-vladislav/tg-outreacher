# Telegram Outreach Platform

Web-приложение для:
- проверки телефонов на регистрацию в Telegram;
- управления сессиями аккаунтов, прокси и fingerprint;
- запуска outreach-кампаний с сообщениями, фоллоуапами и сбором ответов.

## Запуск

1. Создайте и активируйте venv.
2. Установите зависимости:

```bash
pip install -r requirements.txt
```

3. Подготовьте переменные окружения:

```bash
cp .env.example .env
```

4. Запустите приложение:

```bash
python app.py
```

По умолчанию сервер стартует на `http://localhost:5001`.
Файл `.env` загружается автоматически при старте.

## Переменные окружения

- `TELEGRAM_API_ID` - Telegram API ID.
- `TELEGRAM_API_HASH` - Telegram API Hash.
- `APP_PASSWORD` - если задан, включается HTTP Basic Auth для всех страниц и API.
- `SECRET_KEY` - секрет Flask.
- `PORT` - порт сервера (по умолчанию `5001`).
- `FLASK_DEBUG` - `1` для debug-режима, иначе production-поведение.

## Основные разделы UI

- `/` - проверка номеров;
- `/accounts` - аккаунты;
- `/proxies` - прокси;
- `/outreach` - аутрич кампании и планировщик.

## Простой сценарий рассылки

1. Создайте кампанию на странице `/outreach`:
- выберите аккаунты;
- введите текст с переменными (`{name}`, `{company}`, `{position}`, `{phone}`);
- задайте лимит/день и задержку.
2. Выберите кампанию и загрузите базу контактов (CSV/Excel).
3. Сохраните расписание (окно времени, дни, лимит в час).
4. Нажмите `Запустить рассылку`.

Планировщик запускается автоматически и отправляет сообщения только в разрешенное расписанием время.

## Если \"запустил, но ничего не отправляется\"

Проверьте:
- заданы `TELEGRAM_API_ID` и `TELEGRAM_API_HASH`;
- у кампании выбраны аккаунты;
- в базе есть контакты со `status = new` и `user_id`;
- текущее время входит в окно расписания.

## База данных

Используется SQLite: `checker.db`.

Приложение автоматически создаёт таблицы при первом запуске.
